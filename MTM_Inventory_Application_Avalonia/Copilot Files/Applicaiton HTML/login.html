Login

Authentication flow and requirements

Login Screen spec

Purpose: authenticate a user before the app becomes fully available.

What the user does

  • Enter your Infor VISUAL username and password (and Site/Domain if required by your environment).
  • Press Enter or click Login.

What the app does

  • Calls IAuthenticationService.AuthenticateAsync. In Development, accepts Admin/Admin and creates a demo session. In Production, validates Infor VISUAL credentials.
  • On success: updates the session and hides the Login overlay, staying on MainView.
  • On failure: opens the Exception Dialog with a friendly message and technical details.

Commands

  • Login: attempts authentication; disables inputs while in progress.
  • Cancel/Exit: closes the app or returns to a safe state (configurable).
