Settings

Purpose and Overview

The Settings view allows users to manage application-owned configuration settings and view environment context. This interface controls local application behavior and does not directly interact with VISUAL systems, focusing on client-side configuration and database connectivity testing.

How to Use

Access Settings

  1. Navigate to Settings from MainView
  2. Review current configuration values
  3. Modify settings as needed
  4. Test database connectivity if required
  5. Save changes or Cancel to revert

Field and Control Descriptions

Field Type Description Editable
Environment Dropdown Development (0) or Production (1) mode Yes
Warehouse ID Text Input Default warehouse for inventory operations (e.g., "002") Yes
App DB Name Read-Only Label Current application database configuration No
Session User Read-Only Label Currently authenticated user No
Site ID Read-Only Label Current site from session context No

Environment Settings

Development Mode

  • Environment (Development/Production): affects behaviors like Dev login and which DB to use for app-owned data. Changing may require a restart.
  • WarehouseId: default warehouse for forms (e.g., “002”).
  • Test App DB: checks connectivity to the application database and shows success/failure.
  • Session: shows current User and Site from the authenticated session.

Commands

  • Cancel
