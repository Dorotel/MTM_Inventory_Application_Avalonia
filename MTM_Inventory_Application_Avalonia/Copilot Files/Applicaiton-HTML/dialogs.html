Supporting Dialogs

Overview

The application includes several supporting dialogs that provide enhanced functionality for location selection, part searching, and error handling. These modal dialogs integrate seamlessly with the main features to improve user productivity and data accuracy.

Location Picker Dialog

Purpose

Browse and filter inventory locations with current balances. Enables users to search and select locations for inventory transfer and work order operations rather than manually typing location codes.

How to Access

  • Click "Show Location Modal" button in Inventory Transfer
  • Available from any feature requiring location input
  • Opens as a modal dialog over the current view

Features and Controls

Control Function Description
Search Text Filter locations Type to filter by location ID or description
Non-Zero Only Filter checkbox Show only locations with available inventory
Current Warehouse Only Filter checkbox Restrict to current warehouse locations
Current Site Only Filter checkbox Restrict to current site locations
Results Grid Location list Displays filtered locations with balances

Result Columns

  • Location ID: Location identifier code
  • Description: Location name/description
  • On-Hand: Current inventory quantity
  • Available: Available quantity (on-hand minus allocated)
  • Status: Active/inactive status

Commands

  • Select: Applies the highlighted location and closes the dialog
  • Cancel: Closes without making changes
  • Double-click row: Same as Select

Keyboard Support

  • Up/Down arrows: Navigate grid rows
  • Enter: Select highlighted location
  • Esc: Cancel and close dialog

Incomplete Part Dialog

Purpose

Resolve incomplete, invalid, or unknown Part IDs by providing search and selection capabilities. Helps users find correct part numbers when they have partial information or need to browse available parts.

How to Access

  • Click "Resolve Incomplete Part" button in Inventory Transfer
  • Click "Resolve Incomplete Part" button in Work Order Transactions
  • Called automatically when part validation fails

Features and Controls

Control Function Description
Search Text Part search input Enter partial part ID or description to search
Results List Part suggestions Shows matching parts based on search criteria
Selected Part Current selection Highlighted part that will be returned

Search Behavior

  • Seed Text: Dialog pre-populated with caller's current input
  • Live Search: Results update as you type
  • Partial Matching: Supports prefix and contains matching
  • Description Search: Can search by part description as well as ID

Result Information

  • Part ID: Complete part number
  • Description: Part description/name
  • Status: Active/inactive indicator
  • Type: Part type classification

Commands

  • Select: Returns the chosen part ID to the calling form
  • Cancel: Closes without selecting a part
  • Double-click part: Same as Select

Keyboard Support

  • Up/Down arrows: Navigate part list
  • Enter: Select highlighted part
  • Esc: Cancel and close dialog
  • Type to search: Filter results in real-time

Integration

  • Accessed via IPartDialogService.PickPartAsync(seed)
  • Returns selected Part ID as string or null if cancelled
  • Calling form updates appropriate field with returned value

Exception Dialog

Purpose

Display user-friendly error messages with detailed technical information for troubleshooting. Provides a consistent interface for all application errors and exceptions.

When It Appears

  • Any unhandled exception in application operations
  • VISUAL connection or authentication failures
  • Business rule validation errors
  • Configuration or setup issues

Dialog Elements

Element Purpose Content
Title Context identification Operation or method where error occurred
Message User-friendly explanation Clear description of what went wrong
Details Expander Technical information Full exception stack trace and debug info
Action Buttons User response options OK, Retry, Cancel, Help (varies by context)

Available Actions

  • OK: Acknowledge error and continue (most common)
  • Retry: Attempt the operation again (for transient errors)
  • Cancel: Return to previous state without action
  • Help: Open context-sensitive help (planned)
  • Copy Details: Copy error information to clipboard (planned)

Keyboard Support

  • Enter: Activate default action (usually OK)
  • Esc: Cancel/close dialog
  • Ctrl + C: Copy details to clipboard (planned)
  • F1: Open help documentation (planned)
  • Tab: Navigate between buttons

Error Categories

  • Configuration: Database.config, DLL references, setup issues
  • Connection: VISUAL database connectivity problems
  • Authentication: User login and permission issues
  • Business Logic: Closed work orders, insufficient inventory
  • Validation: Invalid data format, missing required fields

Technical Implementation

  • Service: IExceptionHandler.Handle(ex, context)
  • ViewModel: ExceptionDialogViewModel
  • View: Views/Dialogs/ExceptionDialog.axaml
  • Logging: Debug output with fallback handling

Dialog Design Principles

Consistency

  • All dialogs follow similar layout and navigation patterns
  • Consistent button placement and keyboard shortcuts
  • Uniform styling and visual appearance

User Experience

  • Modal dialogs prevent interaction with parent window
  • Clear visual indicators for required actions
  • Responsive layout that works with different screen sizes
  • Accessible via keyboard as well as mouse

Data Integration

  • Dialogs integrate seamlessly with calling forms
  • Selected values automatically populate parent fields
  • Validation occurs before dialog closure
  • Cancel operations leave parent form unchanged

Future Enhancements

Planned Dialog Improvements

  • Enhanced Search: More sophisticated filtering and sorting options
  • Favorites: Save frequently used locations and parts
  • Recent Items: Quick access to recently selected items
  • Bulk Operations: Support for selecting multiple items

Additional Dialogs

  • Advanced Filters: Complex filtering dialogs for data searches
  • Report Parameters: Dialogs for configuring reports and exports
  • Preferences: User-specific settings and customizations
  • About/Help: Application information and help system

Troubleshooting

Common Dialog Issues

  • Dialog not opening: Check for exceptions in debug output
  • Empty results: Verify data connections and filters
  • Selection not working: Ensure proper event handling and data binding
  • Keyboard navigation broken: Check tab order and focus management
