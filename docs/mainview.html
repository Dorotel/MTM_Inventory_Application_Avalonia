Main View (Home)

Purpose and Overview

MainView serves as the central hub after successful login, providing navigation to all inventory features. It hosts feature views in a content region and manages the Login overlay during startup. The view automatically adjusts window sizing for smooth transitions between login and main content.

How to Use

Initial Startup

  1. Application starts showing Login overlay in MainView
  2. After successful authentication, overlay disappears
  3. MainView displays with session information and navigation options

Feature Navigation

  1. Click on a feature button (Inventory Transfer, Work Order Transactions, etc.)
  2. Feature view loads in the main content area
  3. Use feature-specific controls to complete tasks
  4. Return to main hub or navigate to other features as needed

Session Information Display

Read-Only Session Labels

Label Description Source
User Currently logged-in username ISessionContext.UserId
Site Current site identifier ISessionContext.SiteId
Warehouse Current warehouse identifier ISessionContext.WarehouseId

Navigation Commands

Feature Access Buttons

Command Description Access Level
Open Inventory Transfer Loads inventory transfer form for location-to-location transfers MaterialHandler, InventorySpecialist, Lead, ReadOnly
Open Work Order Transaction Loads work order transaction form for issue/receipt operations MaterialHandler, InventorySpecialist, Lead, ReadOnly
Open Settings Loads application settings and configuration All users
Logout Clears session and returns to Login overlay All users

Roles and Permissions

User Roles

  • MaterialHandler: Full access to inventory transfers and work order transactions
  • InventorySpecialist: Full access to all inventory features
  • Lead: Full access to all features with oversight capabilities
  • ReadOnly: Can view features but posting operations may be restricted

Role-Based Feature Access

  • Feature buttons are enabled/disabled based on user role
  • ReadOnly users can access features but posting is disabled within feature flows
  • Unauthorized access attempts are handled by the centralized exception system

Window and Layout Management

Login Overlay Behavior

  • During Login: Window sizes to fit the Login overlay
  • After Login: Window restores previous size or sizes to MainView content
  • Logout: Returns to Login overlay and adjusts window size accordingly

Content Hosting

  • Feature views load in the main content region
  • Only one feature view is active at a time
  • Navigation between features replaces the current content

Keyboard Shortcuts

  • Alt + I: Open Inventory Transfer
  • Alt + W: Open Work Order Transaction
  • Alt + L: Logout

Error Handling

  • All navigation operations use try/catch blocks
  • Errors are routed through IExceptionHandler for consistent display
  • Session issues automatically return to Login overlay
  • Feature loading errors display in the centralized exception dialog

Technical Implementation

Key Components

  • View: Views/MainView.axaml
  • ViewModel: ViewModels/MainViewModel.cs
  • Navigation Service: INavigationService
  • Session Context: ISessionContext

Architecture

  • Single MainView instance hosted in MainWindow
  • Login overlay managed via IsLoginVisible property
  • Feature views injected into CurrentView property
  • Session context propagated to all child features
