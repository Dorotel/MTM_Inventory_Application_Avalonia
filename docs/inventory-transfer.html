Inventory Transfer

Purpose and Overview

The Inventory Transfer feature enables location-to-location inventory transfers within the same warehouse. It provides validation, availability checking, and posting capabilities optimized for keyboard and barcode scanner workflows.

How to Use

Step-by-Step Transfer Process

  1. Access Inventory Transfer from MainView
  2. Enter or scan the Item/Part ID
  3. Click Validate Item to verify the part exists and is active
  4. Enter the From Location (where inventory currently exists)
  5. Enter the To Location (destination location)
  6. Enter the Quantity to transfer
  7. Click Check Availability to verify sufficient inventory
  8. Review the information and click Post Transfer
  9. Use Reset to clear the form for another transfer

Field and Control Descriptions

Field Type Description Required
Item/Part ID Text Input Part number to transfer Yes
Quantity Numeric Input Amount to transfer (must be > 0) Yes
From Location Text Input Source location with available inventory Yes
To Location Text Input Destination location (must differ from source) Yes
Warehouse ID Read-Only Label Current warehouse (typically "002") -
Site ID Read-Only Label Current site identifier -

Validation and Error Handling

Required Fields

  • Item/Part ID: Must exist and be active in VISUAL
  • Quantity: Must be greater than zero
  • From Location: Must exist and contain available inventory
  • To Location: Must exist and differ from From Location

Business Rules

  • Both locations must belong to the current warehouse
  • From Location must have sufficient available quantity
  • Part must be active and transferable
  • Lot/serial controlled items require additional capture (future enhancement)

Common Error Scenarios

  • Part not found: Invalid or inactive part number
  • Insufficient inventory: Not enough available at From Location
  • Invalid location: Location doesn't exist or inactive
  • Same location error: From and To locations cannot be identical
  • Warehouse mismatch: Locations not in current warehouse

Command Descriptions

Command Function When to Use
Validate Item Verifies part exists and is active After entering Item/Part ID
Check Availability (From) Displays available inventory at From Location Before posting to verify sufficient quantity
Show Location Modal Opens Location Picker dialog When you need to search for locations
Resolve Incomplete Part Opens Part search dialog When part ID is incomplete or unknown
Post Transfer Executes the inventory transfer After all validation passes
Reset Clears all fields To start a new transfer

Dialog Integration

Location Picker Dialog

  • Browse available locations with filtering options
  • Filter by current warehouse, current site, or non-zero balances
  • Search by location name or description
  • Select location and return to main form

Incomplete Part Dialog

  • Search for parts by partial ID or description
  • Shows suggested matches based on input
  • Select part and return ID to main form

Keyboard and Scanner Support

Barcode Scanner Integration

  • Item/Part ID field supports barcode input
  • Location fields accept scanned location barcodes
  • Scanner input automatically moves focus to next field

Keyboard Navigation

  • Tab: Move between fields in logical order
  • Enter: Activate focused button
  • Esc: Cancel dialogs or clear current field

Results Display

Availability Check Results

When checking availability, the results grid shows:

  • On-hand quantity at the From Location
  • Allocated quantity (reserved for other operations)
  • Available quantity (on-hand minus allocated)
  • Location details and status

Technical Implementation

Key Components

  • View: Views/InventoryTransferView.axaml
  • ViewModel: ViewModels/InventoryTransferViewModel.cs
  • Service Integration: IInventoryService (planned)
  • Dialog Services: IPartDialogService

VISUAL Integration

  • Uses InventoryTransaction for posting transfers
  • Validates parts via GeneralQuery
  • Checks location balances through VISUAL APIs
  • Maintains TRACE requirements for tracked items

Troubleshooting

Common Issues

  • Scanner not working: Ensure focus is on correct input field
  • Part validation fails: Check part number formatting and status
  • Location errors: Verify locations exist in current warehouse
  • Transfer posting fails: Check VISUAL connection and permissions
