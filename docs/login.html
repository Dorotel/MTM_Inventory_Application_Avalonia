Login

Authentication flow and requirements

Purpose and Overview

The Login screen appears as an overlay inside MainView when the application starts. It authenticates users with Infor VISUAL credentials before allowing access to inventory features. After successful login, the overlay hides and the user remains in MainView.

How to Use

Step-by-Step Workflow

  1. When the application starts, the Login overlay appears centered in MainView
  2. Enter your Infor VISUAL username in the Username field
  3. Enter your password in the Password field (characters are masked)
  4. Optionally enter Site/Domain information if required by your environment
  5. Click Login or press Enter to authenticate
  6. If successful, the overlay disappears and you see the main application interface
  7. If authentication fails, an error dialog provides details about the issue

Field and Control Descriptions

Field Type Description Required
Username Text Input Your Infor VISUAL username Yes
Password Password Input Your Infor VISUAL password (masked) Yes
Site/Domain Text Input Optional site or domain information No
Login Button Button Initiates authentication process -
Cancel Button Button Exits the application -

Validation and Error Handling

Required Fields

  • Username: Must not be empty
  • Password: Must not be empty

Validation Rules

  • Login button is disabled while authentication is in progress
  • Input validation occurs when Login is clicked
  • Empty required fields trigger validation errors

Error Scenarios

  • Missing credentials: "Username is required" or "Password is required"
  • Authentication failure: "Login failed. Check credentials."
  • Connection issues: VISUAL server connection problems
  • Configuration errors: Missing Database.config or DLL references

Development vs Production Authentication

Development Environment

  • Username: Admin
  • Password: Admin
  • Creates a demo session for testing purposes

Production Environment

  • Validates against Infor VISUAL server
  • Uses Database.config for connection settings
  • Supports both OpenLocal and OpenLocalSSO authentication

Keyboard Shortcuts

  • Enter: Submit login (when fields are valid)
  • Esc: Cancel login and exit application

Navigation

  • Successful Login: Overlay hides, MainView becomes active
  • Cancel/Exit: Application closes
  • Error: Exception dialog opens with details

Technical Implementation

Key Components

  • View: Views/Dialogs/LoginView.axaml
  • ViewModel: ViewModels/Dialogs/LoginViewModel.cs
  • Service: IAuthenticationService with placeholder implementation

Business Rules

  • Authentication operations use short-lived VISUAL licenses
  • Passwords are never stored on disk
  • Session context is maintained in memory only
  • All errors route through centralized exception handling

Troubleshooting

Common Issues

  • Cannot connect to VISUAL: Check Database.config file location and contents
  • Authentication repeatedly fails: Verify username/password in VISUAL system
  • Application won't start: Ensure required DLLs are present (LsaCore.dll, LsaShared.dll)
